## Aufgabe 10.1 & 10.2: Floyd-Warshall-Algorithmus

Berechnet kürzeste Pfade zwischen allen Knotenpaaren, erlaubt Umwege über k in Reihenfolge A, B, C, D.

**Graph (ungerichtet, gewichtet):**
Knoten: A, B, C, D
Kanten: (A,B)=9, (A,D)=4, (B,D)=6, (B,C)=1, (C,D)=3

### Initialisierung (D0 und Π0)

Nur direkte Nachbarn eingetragen.

**Kostenmatrix D0:**
| | A | B | C | D |
|---|---|---|---|---|
| **A** | 0 | 9 | ∞ | 4 |
| **B** | 9 | 0 | 1 | 6 |
| **C** | ∞ | 1 | 0 | 3 |
| **D** | 4 | 6 | 3 | 0 |

**Nachfolgermatrix Π0:**
(Zielknoten ist direkter Nachfolger)
| | A | B | C | D |
|---|---|---|---|---|
| **A** | - | B | - | D |
| **B** | A | - | C | D |
| **C** | - | B | - | D |
| **D** | A | B | C | - |

---

### Schritt 1: Umweg über A (k=A)

Prüfe: Ist Weg(i → A → j) besser als Weg(i → j)?
A verbindet B und D (9+4=13), direkte B-D (6) billiger. Keine Änderung.

**Kostenmatrix D1 (keine Änderung):**
| | A | B | C | D |
|---|---|---|---|---|
| **A** | 0 | 9 | ∞ | 4 |
| **B** | 9 | 0 | 1 | 6 |
| **C** | ∞ | 1 | 0 | 3 |
| **D** | 4 | 6 | 3 | 0 |

---

### Schritt 2: Umweg über B (k=B)

Prüfe Verbindungen über B (A, C, D).
Neue Pfade:

- A → B → C: 9 + 1 = 10 (besser als ∞)
- C → B → A: 1 + 9 = 10 (besser als ∞)
- C → B → D: 1 + 6 = 7 (schlechter als 3)

**Kostenmatrix D2:**
| | A | B | C | D |
|---|---|---|---|---|
| **A** | 0 | 9 | **10** | 4 |
| **B** | 9 | 0 | 1 | 6 |
| **C** | **10** | 1 | 0 | 3 |
| **D** | 4 | 6 | 3 | 0 |

**Nachfolgermatrix Π2:**

- Für A → C: Erst nach B (Π[A][C] = Π[A][B] = B)
- Für C → A: Erst nach B (Π[C][A] = Π[C][B] = B)

|       | A     | B | C     | D |
|-------|-------|---|-------|---|
| **A** | -     | B | **B** | D |
| **B** | A     | - | C     | D |
| **C** | **B** | B | -     | D |
| **D** | A     | B | C     | - |

---

### Schritt 3: Umweg über C (k=C)

Prüfe Verbindungen über C (B, D, A via B).

- B → C → D: 1 + 3 = 4 (besser als 6). Update!
- A → C → D: 10 + 3 = 13 (schlechter als 4)
- A → C → B: 10 + 1 = 11 (schlechter als 9)

**Kostenmatrix D3:**
| | A | B | C | D |
|---|---|---|---|---|
| **A** | 0 | 9 | 10 | 4 |
| **B** | 9 | 0 | 1 | **4** |
| **C** | 10 | 1 | 0 | 3 |
| **D** | 4 | **4** | 3 | 0 |

**Nachfolgermatrix Π3:**

- Für B → D: Erst nach C (Π[B][D] = Π[B][C] = C)
- Für D → B: Erst nach C (Π[D][B] = Π[D][C] = C)

|       | A | B     | C | D     |
|-------|---|-------|---|-------|
| **A** | - | B     | B | D     |
| **B** | A | -     | C | **C** |
| **C** | B | B     | - | D     |
| **D** | A | **C** | C | -     |

---

### Schritt 4: Umweg über D (k=D)

Prüfe Verbindungen über D (A, B via C, C).

- A → D → C: 4 + 3 = 7 (besser als 10). Update!
- A → D → B: 4 + 4 = 8 (besser als 9). Update!

**Kostenmatrix D4 (Final):**
| | A | B | C | D |
|---|---|---|---|---|
| **A** | 0 | **8** | **7** | 4 |
| **B** | **8** | 0 | 1 | 4 |
| **C** | **7** | 1 | 0 | 3 |
| **D** | 4 | 4 | 3 | 0 |

**Nachfolgermatrix Π4 (Final):**

- Für A → C: Erst nach D (Π[A][C] = Π[A][D] = D)
- Für A → B: Erst nach D (Π[A][B] = Π[A][D] = D)

|       | A     | B     | C     | D |
|-------|-------|-------|-------|---|
| **A** | -     | **D** | **D** | D |
| **B** | **C** | -     | C     | C |
| **C** | **D** | B     | -     | D |
| **D** | A     | C     | C     | - |

## Aufgabe 10.3: Dijkstra-Algorithmus

**Startknoten:** A
**Annahmen zu unleserlichen Gewichten:**

- A → E = 4
- B → D = 2

**Verlauf:**

1. **Start:** A=0, Rest ∞.
2. **Besuche A (0):**
    - Nachbar B: 0+2=2 (Vorgänger A)
    - Nachbar C: 0+7=7 (Vorgänger A)
    - Nachbar E: 0+4=4 (Vorgänger A)
    - Queue: [B(2), E(4), C(7)]
3. **Besuche B (2):**
    - Nachbar C: 2+3=5. Besser als 7! Update C=5 (Vorgänger B).
    - Nachbar D: 2+2=4. Update D=4 (Vorgänger B).
    - Queue: [D(4), E(4), C(5)]
4. **Besuche D (4):** (oder E, da gleich)
    - Nachbar G: 4+1=5. Update G=5 (Vorgänger D).
    - Nachbar F: 4+2=6. Update F=6 (Vorgänger D).
    - Queue: [E(4), C(5), G(5), F(6)]
5. **Besuche E (4):**
    - Nachbar G: 4+7=11. Schlechter als 5. Kein Update.
    - Queue: [C(5), G(5), F(6)]
6. **Besuche C (5):**
    - Nachbar D: 5+1=6. Schlechter als 4.
    - Nachbar E: 5+2=7. Schlechter als 4.
    - Queue: [G(5), F(6)]
7. **Besuche G (5):**
    - Nachbar F: 5+2=7. Schlechter als 6.
    - Queue: [F(6)]
8. **Besuche F (6):** Ende.

**Ergebnistabelle:**

| Knoten | Kosten | Vorgänger |
|:------:|:------:|:---------:|
|   A    |   0    |     -     |
|   B    |   2    |     A     |
|   C    |   5    |     B     |
|   D    |   4    |     B     |
|   E    |   4    |     A     |
|   F    |   6    |     D     |
|   G    |   5    |     D     |

## Aufgabe 10.4: A*-Algorithmus

**Start:**

```
3 8 4
1 6 2
_ 7 5
```

**Ziel:**

```
1 2 3
8 _ 4
7 6 5
```

**Heuristik h(n):** Manhattan-Distanz
Summe Distanzen aller Steine (außer Leerstelle) zur Zielposition.

**Berechnung h(Start):**

- 1: (1,0) → (0,0) = 1
- 2: (2,1) → (1,0) = 2
- 3: (0,0) → (2,0) = 2
- 4: (2,0) → (2,1) = 1
- 5: (2,2) → (2,2) = 0
- 6: (1,1) → (1,2) = 1
- 7: (1,2) → (0,2) = 1
- 8: (1,0) → (0,1) = 2
- Gesamt h(Start) = 1+2+2+1+0+1+1+2 = 10

**Schritt 1:**
Expandiere Start (g=0, h=10, f=10).
Leerstelle bei (0,2). Mögliche Züge:

1. **7 nach links** (Leerstelle nach rechts):
    - Neue: `3 8 4 / 1 6 2 / 7 _ 5`
    - g=1
    - h sinkt um 1 → h=9
    - f=1+9=10

2. **1 nach unten** (Leerstelle nach oben):
    - Neue: `3 8 4 / _ 6 2 / 1 7 5`
    - g=1
    - h steigt um 1 → h=11
    - f=1+11=12

**Entscheidung:**
Wähle Zug 1 (f=10).
