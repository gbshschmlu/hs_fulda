Schleifeninvarianten verstehen

Was ist eine Invariante?

Eine Schleifeninvariante ist eine Bedingung/Aussage, die:

  1. Vor der ersten Iteration der Schleife gilt (wahr ist)
  2. Nach jeder Iteration weiterhin gilt
  3. Wenn die Schleife endet, uns etwas Nützliches über das Ergebnis sagt

Einfach gesagt: Es ist eine Aussage, die während der gesamten Schleife IMMER wahr bleibt. Das ist wie ein Sicherheitsnetz - wenn die Invariante gilt, wissen wir, dass unser Algorithmus korrekt arbeitet.

Die drei Schritte eines Invarianten-Beweises

1. Initialisierung

Zeige: Die Invariante ist VOR der ersten Iteration wahr

→
2. Erhaltung

Zeige: Wenn die Invariante vor einer Iteration gilt, gilt sie auch danach

→
3. Terminierung

Nutze die Invariante + Abbruchbedingung um das korrekte Ergebnis zu zeigen

Beispiel 1: Einfache Summe

Der Code

sum = 0 for i in range(1, n+1): sum = sum + i # Was ist sum jetzt?

Invariante

Wir behaupten: sum = 1 + 2 + 3 + ... + (i-1) nach jeder Iteration

Oder anders formuliert: sum = Summe der Zahlen von 1 bis (i-1)

Schritt 1: Initialisierung

Vor der ersten Iteration:

  • sum = 0
  • i = 1
  • Behauptung: sum = Summe von 1 bis (i-1) = Summe von 1 bis 0 = 0 ✓

Die Invariante gilt! (Summe von "nichts" ist 0)

Schritt 2: Erhaltung

Annahme: Die Invariante gilt zu Beginn einer Iteration

Zu zeigen: Sie gilt auch am Ende der Iteration

Vorher: sum = 1 + 2 + ... + (i-1)

In der Schleife: sum = sum + i

Nachher: sum = 1 + 2 + ... + (i-1) + i

Das ist genau die Summe von 1 bis i!

In der nächsten Iteration ist i dann i+1, also gilt die Invariante weiter ✓

Schritt 3: Terminierung

Schleife endet bei: i = n+1 (also nach i = n)

Invariante sagt: sum = 1 + 2 + ... + (i-1) = 1 + 2 + ... + n

Ergebnis korrekt! sum enthält die Summe aller Zahlen von 1 bis n ✓

Interaktive Demo: Summe 1 bis n

Beispiel 2: Binäre Exponentiation (aus deiner Aufgabe)

Der Code

x = 2 y = n z = 1 while y > 0: if y % 2 == 1: z = z * x y = y - 1 else: x = x * x y = y / 2 # z enthält jetzt 2^n

Invariante

z * x^y = 2^n und y >= 0

Diese Aussage bleibt durch die ganze Schleife konstant!

Schritt 1: Initialisierung
x = 2
y = n
z = 1

Check: z * x^y = 1 * 2^n = 2^n ✓

Check: y = n >= 0 ✓

Schritt 2: Erhaltung

Fall A: y ist ungerade

Vorher: z * x^y = 2^n

Änderungen: z = z * x, y = y - 1

Nachher: (z * x) * x^(y-1) = z * x^y = 2^n ✓

Fall B: y ist gerade

Vorher: z * x^y = 2^n

Änderungen: x = x * x, y = y / 2

Nachher: z * (x * x)^(y/2) = z * x^y = 2^n ✓

In beiden Fällen bleibt die Invariante erhalten!

Schritt 3: Terminierung

Schleife endet bei: y = 0

Invariante sagt: z * x^0 = z * 1 = z = 2^n

Perfekt! z enthält 2^n ✓

Interaktive Demo: Binäre Exponentiation

Beispiel 3: Insertion Sort

Der Code (vereinfacht)

for i in range(1, len(arr)): # Füge arr[i] an der richtigen Stelle # in arr[0..i-1] ein ...

Invariante

arr[0..i-1] ist sortiert

Der Teil links von i ist immer sortiert!

Schritt 1: Initialisierung

i = 1: arr[0..0] ist nur ein Element → automatisch sortiert ✓

Schritt 2: Erhaltung

Vorher: arr[0..i-1] sortiert

In der Schleife: Füge arr[i] an richtiger Stelle in arr[0..i-1] ein

Nachher: arr[0..i] sortiert ✓

Schritt 3: Terminierung

Schleife endet bei: i = len(arr)

Invariante sagt: arr[0..len(arr)-1] ist sortiert

Das ist das ganze Array! ✓

Warum sind Invarianten wichtig?

1. Korrektheitsbeweise

Mit Invarianten können wir mathematisch beweisen, dass ein Algorithmus korrekt ist - nicht nur durch Tests!

2. Debugging

Wenn du eine Invariante hast, kannst du in jeder Iteration prüfen: Gilt sie noch? Wenn nein → Bug gefunden!

# Python: Assertions nutzen! for i in range(1, len(arr)): # Invariante: arr[0..i-1] ist sortiert assert all(arr[j] <= arr[j+1] for j in range(i-1)), "Invariante verletzt!" # ... rest des Codes

3. Code verstehen

Eine gute Invariante ist wie ein Kommentar, der dir sagt: "Das hier bleibt immer wahr".

4. Optimierung

Wenn du weißt, was invariant bleibt, kannst du Optimierungen machen ohne die Korrektheit zu verletzen.

Pro-Tipps für eigene Invarianten

Wie finde ich eine gute Invariante?

  1. Überlege: Was weiß ich nach jeder Iteration sicher?
  2. Schau auf Variablen: Welche Beziehung zwischen Variablen bleibt konstant?
  3. Denk rückwärts: Was muss am Ende gelten? Was brauchst du dafür während der Schleife?
  4. Sei spezifisch: "arr ist teilweise sortiert" ist besser als "arr ist okay"
